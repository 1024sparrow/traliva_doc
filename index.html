Спецификация фреймворка Тралива

Версия 2

ИЗменения относительно предыдущей версии спецификации

В состав репозитория traliva должны входить скрипты init.sh и build.sh для инициализации и сборки проекта соответственно.

После запуска скрипта init.sh в вышележащей директории должны появиться следующие директории (у пользователя будет запрошено название проекта - его будем обозначать xxx):

  • xxx_doc/ - директория для спецификации проекта xxx
  • xxx_kit/ - набор компонентов для проекта xxx
  • xxx/ - исходники проекта xxx
  • compiled/ - результаты сборки проекта xxx под разные платформы
  • traliva/ - останется директория с выкачанным репозиторием фреймфорка Тралива
  • build.sh - скрипт для сборки (сиволическая ссылка на скрипт в директории traliva)

После инициализации проекта в корне вышележащей директории должна появиться ссылка на собирающий скрипт. Применение каких-либо изменений проекта пересборки невозможны, даже в версии _static - отладочной версии для запуска непосредственно в браузере, без использования какого-либо сервера.
