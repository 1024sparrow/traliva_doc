Пользовательские контейнеры

Данный лэйаут представляет собой контейнер, который определяет пользователь. Представляется объектом с обязательными свойствами type, id и children. Также этот объект должен иметь те свойства, которые необходимы указанному пользователем классу контейнера.

Свойство type должно иметь значение 'container'. По этому свойству фреймворк определяет тип элемента лэйаута.

Свойство id должно задавать уникальный идентификатор виджета, по которому в свойстве widgets фреймворк будет запрашивать информацию о конструкторе и объекте состояния

Имена полей, по которым будут формироваться дочерние виджеты, определяются в классе виджета-контейнера, который указывается в свйостве widgets (должен быть отнаследован от класса WidgetStateSubscriber).

Пример (пусть это будет что-то вроде лэйаута strip с дополнительными основным и дополнительными тулбарами):

layouts: { my_layout: { ... { type: 'container', id: 'myContainerLikeStrip', toolbar: 'const myStripToolbar', addonToolbar: { type: 'strip', orient: 'v', items: [...] }, orient: 'h', items: [ 'my_beautifulWidget', { 'type': 'strip', 'orient': 'v', 'items': [...] }, { 'type': 'stack', ... }, 'const myAnotherBeautifulWidget' ] } ... } } ... widgets: { ... myContainerLikeStrip: { constructor: TralivaKit.LikeStripContainer, options: {type: 'compact'}, substate: '...' }, ... }, ...

В этом случае класс LikeStripContainer (в пространстве имён TralivaKit) должен иметь следующий статический метод:

LikeStripContainer.widgetsFields = function(){ return ['toolbar', 'addonToolbar', 'items']; }
